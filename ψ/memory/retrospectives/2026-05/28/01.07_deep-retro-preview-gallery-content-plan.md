## Key Code Changes

### 1. Python SSE Preview Server (`~/preview/.server.py`)
- Architecture: `watchdog` file watcher → SSE push → browser auto-refresh  
- Poll interval: 2 seconds  
- Port: 8081  
- Pattern: start server → scan `~/preview/` → generate gallery HTML → push SSE event on change

### 2. launchd plist (`com.uficon.previewserver.plist`)
- `ProgramArguments`: `python3 ~/preview/.server.py`
- `RunAtLoad`: true  
- `KeepAlive`: true  
- ⚠️ `launchctl load` ยังไม่ verified post-reboot

### 3. Content Plan HTML Generator (3-part split)
- `june_items_1_30.py` → W1–W3 items (30 items)
- `june_items_31_60.py` → W3–W5 items (30 items)

- W2 มี 11 items (ควรเป็น 12) — off-by-one, total ยังครบ 60

## Architecture Decisions

1. **SSE over WebSocket**: เลือก SSE เพราะ one-directional push ง่ายกว่า และ native browser support ไม่ต้องมี library เพิ่ม
2. **launchd over cron**: launchd ให้ `KeepAlive` auto-restart ถ้า server crash, cron ไม่มี feature นี้
3. **Split Python scripts**: แทนที่จะเขียน 60 items ใน file เดียว แบ่งเป็น 3 files เพื่อหลีกเลี่ยง token limit cutoff กลางงาน
4. **Platform-dedicated content**: ULite/Thisshop/Ufund มี content series ของตัวเอง (ไม่ใช่ repost จาก iStudio) เพื่อตอบโจทย์ audience แต่ละ platform

## Lessons Learned (Multi-Pattern — Deep Mode)

### L1: Claude Background Process ≠ Persistent Service [High Confidence]
ทุก process ที่รันด้วย `&` หรือ background shell ใน Claude จะตายเมื่อ context compress, session timeout, หรือ user ปิด Claude ต้องใช้ launchd (macOS) หรือ systemd (Linux) สำหรับ service ที่ต้องอยู่ข้าม session

### L2: launchd Script ห้ามอยู่ใน /tmp/ [High Confidence]
`/tmp/` ถูก clear เมื่อ reboot — plist ที่ชี้ไปยัง `/tmp/script.py` จะ register ได้แต่ fail ทันทีหลัง restart permanent scripts: `~/preview/.server.py`, `~/bin/`, หรือ `~/.local/bin/`

### L3: Content Plan >35 items ต้องแบ่ง Python script [High Confidence]
60 items × (caption + graphic brief) เกิน token output limit ใน single generation แบ่งเป็น 3 ไฟล์ (part1 + part2 + builder) เป็น pattern ที่ใช้ซ้ำได้สำหรับทุก content plan ขนาดใหญ่

### L4: Platform Dedicated Content > Cross-Posting [Medium Confidence]
ULite/Ufund/Thisshop ต้องมี series ของตัวเอง ไม่ใช่แค่ tag ชื่อ platform ท้ายโพสต์ iStudio สอนสิ่งนี้และ Beejeni confirm ว่านี่คือ direction ที่ถูกต้อง

### L5: /rrr ต้องรันก่อน Session ปิด [High Confidence]
Session fb7f7c26 เป็นหลักฐานว่า session split ทำให้ retro ขาด substance การรัน /rrr หลัง reopen ทำให้เสีย context และ session JSONL ว่างเปล่า Rule: /rrr ก่อนปิด ไม่ใช่หลัง reopen
