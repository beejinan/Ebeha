## Honest Feedback

**Friction 1 — CRM segment instability**: Three rebuilds of the same section in one session is too many. The root cause was that the segment framing wasn't locked before building. Product-first felt natural at start but Beejeni had occupation segments in mind from the beginning. If I had asked "should this be by product or by customer type?" upfront, we'd have saved one full iteration. Lesson: segment definition before structure.

**Friction 2 — Thai character string matching**: The Edit tool failed on the KOL PPTX update because Thai text in `old_string` didn't exactly match the file. This is a real friction point in Thai-language files: copying from rendered HTML vs raw file content can diverge subtly. Need to always grep-verify exact text before constructing Edit calls.

**Friction 3 — Agent 3 (dig timeline) permission gap**: The dig agent needed Bash permission, hit the approval prompt, and couldn't complete its task. Session dig data was lost. This is a structural gap: `/rrr --deep` with Bash-requiring subagents should either pre-authorize or fall back gracefully. The timeline was reconstructed from memory instead — worked out, but at a cost.

## Lessons Learned

1. **Lock segments before slides**: CRM/audience framing must be decided before building structure. Ask "by product or by persona?" before row 1 exists.
2. **Grep before Edit in Thai files**: Thai characters can cause silent mismatches in string matching. Always grep exact lines first.
3. **Thai accordion pattern is now standard shell**: `explain-btn` + `explain-box` + `toggleExplain()` works. Use it for all future briefing HTML files.
4. **Medical/education professionals as a standing segment**: Appears in BTS (บุคลากรแพทย์/การศึกษา) and KOL pivot. Likely Beejeni's emerging focus audience beyond students.
