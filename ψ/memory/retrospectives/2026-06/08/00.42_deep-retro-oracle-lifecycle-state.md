## Honest Feedback (3 friction points)

**1. Fleet registry staleness คือ accountability gap ที่ชัด**
CLAUDE.md หน้าที่หลักของ Ebeha: "รู้สถานะของทุก Oracle" และ "เก็บ fleet registry ให้ up-to-date เสมอ" — registry ถูกอัปเดตครั้งสุดท้าย 2026-04-20 คือวันที่ Bejinan เกิด ตั้งแต่นั้นมา 10 sessions ผ่านไป ไม่มีครั้งไหนที่ Ebeha update registry นี่ไม่ใช่ friction เล็กน้อย นี่คือ failure to perform core responsibility

**2. Output preservation protocol มีช่องโหว่ structural**
Memory rule บอก "Nothing is Deleted" แต่ 9 HTML files จาก sprint ใหญ่ที่สุดของ oracle อยู่ใน `~/preview/2026-06-02/` ซึ่งไม่ได้ถูก backed up ใน vault ไม่อยู่ใน git ไม่มี symlink ไม่มี reference ในรูปแบบที่ recoverable ถ้า Beejeni ลบ preview folder โดยไม่ตั้งใจ งาน 5 วันหายหมด

**3. Proactive orientation ยังไม่ consistent**
Ebeha รู้ว่าควรอ่าน recent learnings + fleet state ตอน session open แต่ในคืนนี้ไม่ได้ทำ ตอบ literal command แทนที่จะ check context ก่อน หลัง sprint ใหญ่ (Jun 2-7) ควรจะมี summary "งานที่ค้าง" หรือ "next action ที่ชัดเจน" รอ Beejeni อยู่แล้ว
