**Date**: 2026-06-02

**Session**: 5183caf1

## Pattern

When Beejeni runs `/clear` followed immediately by `/rrr`, this is a deliberate session reset signal — not an error or oversight.

**What it means**:
- Fresh context wanted
- Previous session truly closed
- New work is about to begin
- No need to reconstruct prior conversation

## What to do after /clear + /rrr

1. Complete the retrospective (even if thin)
2. Re-orient by reading fleet registry + recent memory index
3. Ask Beejeni what's on deck for today
4. Don't assume continuity from prior session — start clean

## What NOT to do

- Don't try to reconstruct what happened before `/clear`
- Don't pad the retrospective with fake detail
- Don't treat this as an error state
